Export Guide

From SelectionTo Implementation

Tama Templates의 목적은 브라우저 안에서 끝나는 것이 아니라, 다른 개발 환경에서 구현을 바로 시작할 수 있는 정리된 handoff를 만드는 것입니다.

MVP Export Files

`composition.json`, `design-template.json`, `layout-template.json`, `implementation-brief.md` 를 기본 handoff bundle로 사용합니다.

Steps

Recommended Handoff Flow

아래 순서를 기준으로 다른 LLM 또는 개발 환경에 넘기면 의도 손실을 줄일 수 있습니다.

1. Design Concept Lock

Basic Foundation Template, PLAT Neo-Brutalist Template, Kawaii Cyber-Pop Motion Reference, Basic x Kawaii Live Switch Proof 중 하나를 먼저 고정합니다. 이것이 스타일 계층입니다.

2. Layout Selection

16:9 레이아웃 패밀리 중 하나와 변형 번호를 고릅니다. 이것이 구조 계층입니다.

3. Export Bundle

`Compose` 에서 `composition.json`, `design-template.json`, `layout-template.json`, `implementation-brief.md` 를 내려받아 구현 요청의 기준점으로 사용합니다.

4. Downstream Prompting

외부 환경에서는 `layout-template.json` 으로 구조를 먼저 구현하고, `design-template.json` 으로 스타일 계층을 입힌 뒤, `composition.json` 과 brief를 최종 지침으로 사용하는 순서를 유지하는 것이 중요합니다.
